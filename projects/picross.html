Picross

This was my final project created in the Intro to Game Web Tech class I took during the first half of my 2nd year in RIT’s Game Design & Development program. Since our time was limited to only a few days to create a simple game using pixi.js, I felt Picross would be simple, albeit engaging enough to suffice.

I used bootstrap to handle the layout of the website, though the creation of the board was entirely custom. One of five possible 16x16 puzzles is selected for the player to attempt, with each puzzle being the sprite of a NES character, those being Mario, Link, Kirby, Goomba, and Waddle Dee respectively.

Though the only puzzles that exist are 16x16, the code written supports puzzles of any given size, as it takes black and white images and converts them into individual tiles. Being able to convert the images into the format necessary to read them and convert them into tiles was the biggest hurdle, as images normally load asynchronously, and I needed them properly loaded prior to creating the board. In the end, I just needed to better utilize promises in order to achieve that.

Ultimately, this was a simple, albeit challenging project all the same. What I needed to do was simple, it was merely getting to that point that proved difficult at times. In spite of that, I was happy to be able to create this, as I’m an avid fan of picross.
