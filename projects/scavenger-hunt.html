Internet Scavenger Hunt

The final project for Rich Media Web Application Development II required us to create a full-stack web application using a MongoDB backend, alongside Redis. Since we had also finished learning React not too long ago, we were encouraged to use it as well, especially since it's extremely useful for creating reusable front-end components.

The application was required to have support for Creating, Reading, Updating, and Deleting (CRUD) user accounts, as well as some type of MongoDB document, and a proof of concept for a payment model. The most common example of this was locking certain features behind a paywall, or limiting the amount of a feature you could utilize without paying for it. As a proof of concept though, premium would just be a toggleable feature stored on a person's account.

At the time, I was thinking about how centralized the modern internet is, in that we typically only visit 4, maybe 5 websites where, back during the days of web 1.0 and web 2.0, you explored so many other different types of sites. The concept of making an internet scavenger hunt was intended to bring some of that back, by forcing people participating in these scavenger hunts to explore and find sites they'd otherwise have no reason to go to, and maybe help to spark some regained interest in de-centralizing the experience of browsing the web.

To do this, I created 4 Mongoose models to be used in tandem with MongoDB: Users, Scavenger Hunts, Items, and Submissions. Users is self explanatory, Scavenger Hunts were the larger scale hunts themselves, items were the collections of tasks that comprised each scavenger hunt, and submissions were the user submissions to a particular scavenger hunt, since the idea was to be able to compete with other users.

The backend gave me some trouble due to my relative inexperience with MongoDB at the time, but nothing too terrible. I also had more trouble than I would've liked with the front-end, as I spent a large amount of time trying to use the React Bootstrap library, only to run into several issues that eventually compelled me to just use standard Bootstrap.

Due to time constraints from other finals and final projects, I wasn't able to finish everything I would've liked to on the project, and was only able to get a rudimentary front-end up. What I had created I was somewhat proud of, but at some point in the future, I think I'd like to reattempt making something like this, since I still think it'd be a neat idea.
