CPS-3 ROM Conversion Tool

This was a Java program I created in March of 2020 in order to make romhacks for Capcom Play System 3 (CPS-3) titles playable on the newly released arcade emulator Final Burn Neo and FightCade 2. Previously, FightCade was built off of FInal Burn Alpha, which used a particular format for its ROMs. When the jump was made to Final Burn Neo, the ROM format, at least for Capcom Play System 3 (CPS-3) titles was changed.

The previous format was known as the combined ROM format, and looked as follows:

10 Essential data (code, attack data, music data, frame data)
20 Essential data cont. (sometimes unused)
30 Audio (instrument samples, voices)
31 Audio cont.
40 Graphics data
41 Graphics data
50 Graphics data
51 Graphics data (sometimes unused)
60 Graphics data (only used in Street Fighter III: 3rd Strike)
61 Graphics data (only used in Street Fighter III: 3rd Strike)

The 10 and 20 files are generally considered data files, where the 30 file onwards are considered user files, as they exist for anything visual or auditory when it comes to these games. What they store is also interchangeable, though all CPS-3 games store their data the same way.

The split ROM format stores data similarly, but expands each individual file into 4 files, referred to as simm files. Below are some examples:

10
x-simm1.0 x-simm1.1 x-simm1.2 x-simm1.3
30
x-simm3.0 x-simm3.1 x-simm3.2 x-simm3.3
31
x-simm3.4 x-simm3.5 x-simm3.6 x-simm3.7

In these cases, x refer to the ROM name, such as jojoba in the case of JoJo's Bizarre Adventure: Heritage for the Future, or sfiii3 in the case of Street Fighter III: 3rd Strike.

It should be noted that in some cases (Street Fighter 3: New Generation and Red Earth/Warzard), the user file may only be split into two simm files rather than 4, though this has only been when the user file is half the maximum possible size. The maximum size of a combined ROM file is 8,388,608 (0x800000) bytes and in the case of Red Earth and New Generation, their respective 50 files are only 4,194,304 (0x400000) bytes in length.

Itâ€™s also important to note that data files and user files are split differently from one another. With the data files, each sequential byte is moved into a different file before looping the process. As an example:

Byte 1 goes in simm1.0

Byte 2 goes in simm1.1

Byte 3 goes in simm1.2

Byte 4 goes in simm1.3

Byte 5 goes in simm1.0

Byte 6 goes in simm1.1

...

When it comes to the user files, the first half of the file is split the same way between the first and second simm files. The latter half of the file is then split between the third and fourth simm files. Again, as an example:

Byte 1 goes in simm3.0

Byte 2 goes in simm3.1

Byte 3 goes in simm3.0

Byte 4 goes in simm3.1

...

Byte 4,194,304 goes in simm3.2

Byte 4,194,305 goes in simm3.3

Byte 4,194,306 goes in simm3.2

Byte 4,194,307 goes in simm3.3

...

This tool was created to handle the conversion process from one format to another, and vice versa, which was especially helpful as the hex editor used to decrypt and edit ROM files (X.C.O.P.Y.) only worked with the combined format.

Originally, this tool only supported JoJoâ€™s Bizarre Adventure: Heritage for the Future, but it was updated in June of 2021 to include support for all CPS-3 titles.
